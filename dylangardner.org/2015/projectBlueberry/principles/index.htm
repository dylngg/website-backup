Principles

  1. Simple, Elegant Code

    2. Web developers have a habit of adding tons of dividers and just making sure it works instead of making it simple. However, I have a different mindset. Instead of wrapping everything in dividers, I only use dividers when I need to actually seperate objects. This has a positive effect on page performance and makes the code more simplisic.

  2. Be Different

    3. It seems nowdays that every site you see follows a few similar characterisics. There's the classic navbar that can stick to the top, then there's the side navigation and a footer at the bottom. These have become so common that html5 even added a element just for developers(aside). But of course, I think differently. I create things that are not ordinary, things that make people realize that it's good to be different.

  3. Fluid Layout on all devices

    4. There are over a thousand different screen sizes in the world today. Creating content for just a the portion of people just sounds crazy. That's why when I build a website, I make sure it works on as many sizes as possible. After all, mobile devices are becoming some of the most popular devices in the world.
